Preguntas

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    Para que nuestra base de datos no se llene de valores basura, valores que no siguen un formato que ya establecimos o que simplemente no nos sirvan.

  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    Solo con usar algunas de las palabras reservadas desde la consola de desarollador en el navegador (Ada, 2020)

  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    Dice que es necesarios porque lo que se debe de hacer es usar tanto la validación desde el lado del cliente como la validación del lado del servidor (Ada, 2020).
